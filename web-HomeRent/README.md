# إيجار ويب (IjarWeb) - نظام كامل (Web + API)

## الأدوار
- site_admin: مدير الموقع
- owner: مالك (الحساب الرئيسي)
- staff: مستخدمون مضافون تحت نفس حساب المالك
- tenant: مستأجر

## أهم الميزات المطلوبة (مُنفّذة)
- المالك يضيف العقار أولاً ثم يضيف الوحدات داخل العقار (شقة/معرض/أو اسم نوع مخصص)
- إنشاء عقد يولّد حساب المستأجر تلقائيًا بكلمة مرور عشوائية تُرسل بالإيميل
- خطط السداد: شهري/دفعة واحدة/دفعتين/ثلاث/أربع + يوم ثابت داخل العقد (due_day)
- المستأجر يرفع طلب اعتماد سداد (ملف واحد: صورة أو PDF) ويُحفظ بتاريخ الرفع
- اعتماد الطلب يولّد سداد تلقائي + سند سداد باسم المستأجر
- المالك يمكنه إضافة سداد يدوي + سند سداد
- تقارير لكل عقار أو لجميع العقارات
- تنبيهات للمالك عبر البريد عند وصول طلب اعتماد جديد
- المالك يضيف مستخدمين (staff) بنفس الحساب للمتابعة
- مدير الموقع: إيقاف/تفعيل المستخدمين، إرسال إشعارات داخل الحساب أو عبر البريد، إحصائيات (عقارات/وحدات/مستخدمين/مستأجرين)

## التشغيل
1) أنشئ DB MySQL ثم نفّذ: db/schema.sql
2) أنشئ مدير الموقع: db/seed_site_admin.sql
3) أنشئ مالك: db/seed_owner.sql
4) عدّل api/_core/config.php (DB + BASE_URL + JWT_SECRET + البريد MAIL_FROM)
5) ارفع مجلد (ايجار-ويب) على السيرفر، وافتح:
   - web/index.html (ثم login.html)

> البريد: النظام يستخدم mail() في PHP. إذا لم يعمل على سيرفرك، فعّل SMTP لاحقًا (PHPMailer).

## تنبيهات الاستحقاق/التأخرات/انتهاء العقود (Cron)
تم إضافة سكربت كرون يومي يقوم بإنشاء تنبيه للمالك + إرسال بريد:
- المتأخرات (أقل من اليوم)
- استحقاقات خلال 30 يوم
- عقود تنتهي خلال 60 يوم

المسار:
- api/cron/daily_notifications.php

التشغيل عبر CLI:
- php api/cron/daily_notifications.php

أو عبر HTTP (اختياري):
- ضع قيمة في CRON_TOKEN داخل api/_core/config.php
- ثم استدع:
  .../api/cron/daily_notifications.php?token=TOKEN
